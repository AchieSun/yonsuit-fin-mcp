## 快速开始

### 1. 环境准备

确保已安装Node.js 20.0.0或更高版本：

```bash
node --version
# v20.x.x
```

### 2. 安装和配置

```bash
# 进入项目目录
cd yonyou-mcp

# 安装依赖
npm install

# 复制环境变量文件
cp .env.example .env

# 编辑.env文件，填入实际配置
# YONYOU_APP_KEY=your_app_key
# YONYOU_APP_SECRET=your_app_secret
# YONYOU_TENANT_ID=your_tenant_id
# YONYOU_DATA_CENTER_DOMAIN=your_domain

# 构建项目
npm run build
```

### 3. 配置AI助手

#### Claude Desktop配置

编辑Claude Desktop配置文件：

**macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`

**Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "yonyou": {
      "command": "node",
      "args": ["/absolute/path/to/yonyou-mcp/dist/index.js"],
      "env": {
        "YONYOU_APP_KEY": "your_app_key",
        "YONYOU_APP_SECRET": "your_app_secret",
        "YONYOU_TENANT_ID": "your_tenant_id",
        "YONYOU_DATA_CENTER_DOMAIN": "your_domain"
      }
    }
  }
}
```

重启Claude Desktop后，即可使用用友做账功能。

## 最佳实践

### 1. 凭证创建最佳实践

**先查询科目是否存在**

在创建凭证前，建议先确认科目编码是否正确：

```
请帮我查询科目1001的详细信息
```

**使用外部凭证号关联**

创建凭证时，建议使用 `externalVoucherNo` 字段关联外部系统的单据号，便于后续查询和追溯：

```json
{
  "voucherTypeCode": "记-1",
  "voucherDate": "2024-01-15",
  "accountingPeriod": "2024-01",
  "externalVoucherNo": "DD-2024-001",
  "entries": [...]
}
```

**验证借贷平衡**

创建凭证时，系统会自动验证借贷平衡，建议在提交前确认：

```
请帮我创建一个凭证，借方10000元，贷方10000元...
```

### 2. 查询优化最佳实践

**使用分页查询**

查询大量数据时，使用分页参数避免一次返回过多数据：

```json
{
  "pageNum": 1,
  "pageSize": 20
}
```

**使用精确查询条件**

尽量使用精确的查询条件，减少返回的数据量：

```json
{
  "accountingPeriod": "2024-01",
  "auditStatus": "unaudited"
}
```

### 3. 错误处理最佳实践

**检查返回结果**

每次操作后检查返回的 `success` 字段：

```json
{
  "success": true,
  "message": "创建凭证成功",
  "data": {...}
}
```

**处理错误信息**

操作失败时，查看错误信息并进行相应处理：

```json
{
  "success": false,
  "message": "创建凭证失败",
  "error": {
    "code": "SAVE_VOUCHER_ERROR",
    "message": "借贷不平衡"
  }
}
```

## 与AI助手交互技巧

### 1. 提供完整信息

创建凭证时，提供完整的信息：

```
请帮我创建一个凭证：
- 凭证类型：记-1
- 凭证日期：2024-01-15
- 会计期间：2024-01
- 摘要：购买办公用品
- 分录：
  借：管理费用-办公费 5000元
  贷：银行存款 5000元
```

### 2. 使用自然语言描述

AI助手可以理解自然语言描述：

```
帮我查一下上个月银行存款花了多少钱
```

AI助手会自动解析并调用相应的工具。

### 3. 分步骤操作

复杂的操作可以分步骤进行：

```
第一步：请帮我查询科目1001的详细信息
第二步：请帮我查询该科目2024年1月的明细账
第三步：请帮我核对余额是否正确
```

### 4. 确认操作

对于重要的操作（如删除凭证），建议先确认：

```
请帮我确认凭证"记-2024-01-001"的详细信息，确认后我再决定是否删除
```

### 5. 使用上下文

AI助手可以记住上下文，进行连续操作：

```
用户：请帮我查询科目1001的详细信息
AI：科目1001是库存现金...
用户：查询这个科目上个月的明细账
AI：好的，查询库存现金科目2024年1月的明细账...
```

## 注意事项

### 1. 数据安全

- 敏感信息（如AppKey、AppSecret）请妥善保管
- 不要在日志中记录敏感信息
- 定期更换API密钥

### 2. 操作权限

- 确保API账号有足够的权限
- 某些操作可能需要审批流程
- 删除操作不可恢复，请谨慎操作

### 3. 数据一致性

- 创建凭证前确认借贷平衡
- 注意会计期间的开启状态
- 已审核/已记账的凭证不能直接修改

### 4. 性能优化

- 大量数据查询时使用分页
- 避免频繁调用API
- 合理使用缓存
